# CalorixDesktop

## Общие понятия для проекта
- виджет (widget) - в qt виджет является всем что можно будет отобразить на экране
это дочерний элемент от элемента QObject который является базовым классом для всех визуальных элементов
в текущем проекте виджет будет обозначать хранилище для нескольких модулей и других виджетов.
Если пояснять на примере, главное окно приложения где могут быть другие виджеты, кнопки и т.п. будет являтся виджетом.
Т.к. он хранит в себе другие виджеты, модули и элементы
- модули (moduls) - модуль в данном проекте будет обозначать какой-то визуальный или не визуальный кусок кода который будет использоваться больше 2 раз
по принципу программирования DRY (don't repeat yourself), все алгоритмы и механизмы отображения что будут повторяться часто будут вынесены как модули или классы
для дальнейшего использования их в коде

На каждый сложный элемент следует создавать свой модуль или виджет, в случае если он использует несколько других
модулей которые ему не обходимы, следует создать отдельную папку с соответствующим названием и писать всю реализацию в разных файлах, дабы избежать
файла с 1000 строчек кода

## Обоснование папок

- `module` - модули питон, функции которые выполняют какую-то работу
- `widget_module` - модули питон для реализации виджетов
- `module_types` - классы контейнеры для передачи информации между модулями или же qml виджетами
- `qml_module` - модули qml которые можно будет использовать в дальнейшем
- `qml_widgets` - виджеты qml для отображения нескольких других виджетов, модулей и элементов
